1. Industrial Context & Research Problem ~2 min

Extended enterprise product development

  • OEMs, suppliers, certification authorities — multi-decade lifecycles
  • PLM standards: ISO STEP AP242 (3D engineering), AP239 (lifecycle support)
  • MBSE: SysML, Arcadia/Capella — structurally isolated from PLM geometric data
  • Sovereign data spaces: Gaia-X, Catena-X, DecadeX — governed sharing via RDF, OWL, SHACL, DCAT, ODRL
The layering problem
Sovereign platforms ensure technical-layer interoperability — not semantic consistency of engineering payloads.
Two partners may exchange STEP AP242 files under identical ODRL policies yet interpret assembly constraints differently.

Two research limitations

L1 — Semantic consistency
Heterogeneous formats (STEP, SysML, ArchiMate) remain semantically opaque in sovereign data space architectures.
L2 — "RDFize everything"
OWL 2 expressiveness–decidability trade-offs → either non-decidable ontologies or semantically lossy translations. Legacy PLM standards cannot be fully absorbed.
Consequence
Legacy PLM standards remain opaque artifacts — governed access without explorable semantics for the architects governing multi-year transformations.

3. Related Work & Identified Gap ~2 min

Interoperability frameworks

  • 36 types, 117 definitions [Maciel, 2023]
  • ATHENA — ontology, MDA, SOA [Berre, 2007]
  • FIF + EH4I — formal interoperability, continuous change [Vernadat, 2025]
Address formal alignment, technical integration, standards governance — not the cognitive challenge of comprehension.

Visualization tools

  • UML/SysML/ArchiMate — static diagrams, tree/canvas separation
  • Linked Data viewers — non-hierarchical "hairball" graphs
  • Cytoscape.js + fCoSE — compound graphs, animated transitions — but no semantic typing
Katifori et al. (2019) · Franz et al. (2016) · Balci & Dogrusoz (2022)
The gap (3 simultaneous requirements):
  • ① Interactive compound graph navigation
  • ② Semantic typing from language specs
  • ③ Multi-standard heterogeneity (STEP + OWL + ArchiMate)
Implicit assumption in current frameworks: stakeholders can mentally construct coherent multi-standard system pictures from fragmented documentation. Prototype evidence suggests this assumption fails — and that the failure is costly.

4. Applied Semantic Cartography — Definition ~1 min 30

Applied Semantic Cartography (ASC):
Semantization of a compound graph by deriving types and rendering rules from a visual language specification, while preserving underlying formal semantics in their native frameworks (OWL axioms, STEP schemas, ArchiMate metamodel).

What ASC is

  • A cognitive layer over heterogeneous formal standards
  • A unified interactive space where evolution pathways become comprehensible to architects
  • An instrument for visual semantic coherence — seeing and interacting with cross-standard relationships

Key distinction

ASC does not replace OWL, STEP, or ArchiMate. It provides the navigational and cognitive infrastructure these standards do not themselves supply.

What ASC is not

  • Not a new semantic standard
  • Not a translation layer or middleware
  • Not a replacement for formal ontological alignment
Theoretical grounding:
Interoperability challenges are not merely technical problems solvable through standardisation, but wicked problems requiring continuous stakeholder alignment — the cognitive dimension must be addressed, not assumed.
~1 min 30. Définition centrale — lire précisément. Insister sur la préservation de la sémantique formelle dans les frameworks natifs. "Wicked problems" peut générer des questions — être prête avec une formulation courte. Transition : architecture technique.

5. Technical Approach & Semantization ~1 min 30

Compound graph foundation (existing)

  • Cytoscape.js — traversal, centrality, clustering, compound node support
  • fCoSE — fast layout with constraint support, animated transitions
  • expand/collapse plugin — selective animated hierarchical navigation

Semantization layer (contribution)

  • Type system derived from visual language specification
  • Rendering rules engine: type → symbol (shape, color, icon, line weight, dash pattern)
  • Graph analytics (Cytoscape.js) + animated compound viz + semantic typing = capability absent from existing tools

From specification to rendering — ArchiMate 3.2

  • Extract node construct types (business-actor, application-component, data-object…)
  • Extract relationship types (association, aggregation, realization, triggering…)
  • Derive rendering rules per type — formal semantics remain in native framework
Generalised to languages without formal visual spec:
OWL/SHACL/DCAT, ISO STEP Part 11 (EXPRESS), Part 21 (exchange format) — feasibility demonstrated in all cases. Identifiers preserved throughout → cross-tool traceability.
Franz et al. (2016) · Balci & Dogrusoz (2022) · Dogrusoz et al. (2009, 2018)

9. Contributions ~1 min 30

Theoretical

T1
Applied Semantic Cartography — a paradigm bridging the cognitive gap in enterprise interoperability transformation; semantic typing as a first-class concern alongside formal semantic alignment
T2
Requalification of the problem — interoperability challenges as wicked problems requiring stakeholder comprehension, not only technical integration; visual semantic coherence equally critical as formal semantic consistency
T3

Multi-standard visualization — STEP AP242, OWL/DCAT, ArchiMate in a single interactive compound graph; semantic types preserved across standard boundaries

10. Limitations, Perspectives & Conclusion ~2 min

Known limitations

Evaluation
Exploratory and qualitative — prototype usage scenarios, not controlled experiment.
Scalability
Acceptable up to ~2,000 nodes. Semantic lenses and multi-view coordination required beyond density thresholds.
Polyglot SC
Feasibility per language separately. Multi-language unified map (incl. ODRL policies) not yet resolved — type system unification required.
STEP coverage
Part 21 exchange instances only. Reference models and EXPRESS schemas not yet addressed.

Future directions

  • Controlled evaluation of navigation effort and misalignment detection
  • Category theory — formal mapping types across cartographies
  • SC of DecadeX for Required PLM Interoperability Assessment
This paper addresses a gap in the interoperability literature: the absence of interactive compound graph visualization for semantic relationships across STEP, OWL/DCAT, and ArchiMate — a methodological blind spot in existing frameworks.
